Todos los formularios en web manejan metodos para el envio de las diferentes peticiones que realiza usuario Debebos de recordar que la mayoria de las aplicaciones en web trabajan sobre una arquitectura cliente servidor. Significa que el cliente realiza una peticion (request) y el servidor debe de antender inmediatanete una respuesta(response). Los metodos mas comunes son GET:

este metodo se encarga de enviar la indormacion del formulario por medio de la salida o url estandar del navegador, esto proporciona que pueda existir inyeccion de codigo realizando violaciones a la seguridad del sistema

POST:

Este metodo se encarga de enciar la informaxion mediante un metodo de ocltamiento. Lo que nos ofrece mayor segurida en el manejo del sistema

GET solo se usa para operaciones de lectura POST sirve para cambiar la informacion de un recurso por parte del servidor(insertar un registro) Delete sirve para eliminar un recuero en particular Put alamcena la informacion de un recurso en particualr para su actualizacion Head regresa el codigo de respueta y cualquier cabecera HTTPS asociad
